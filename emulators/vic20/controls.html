Commodore · VIC-20

Commodore VIC-20 — Controls

Self-hosted VICE xvic (the VIC-20 build of the gold-standard Commodore emulator) running in your browser via EmulatorJS. Most VIC-20 software is cartridge-based and boots instantly; the few disk-based titles autoboot when the page loads. Click the canvas first so the emulator receives keyboard input.

The bundled VIC-20 is configured with full memory expansion (35K — all RAM banks at $2000-$7FFF + $A000-$BFFF). Cartridge games override this with their own memory map; disk and tape games written for any of the standard +3K / +8K / +16K / +24K configurations all run cleanly under +all.

When a disk game is loading, the initial read takes around 20 seconds — that's not a freeze, it's the real-time 1540 drive emulation reading the disk over the IEC bus at period-authentic speed. You'll hear the drive chunk and stepper-click the whole way through; the game starts when it falls silent.

VIC-20 keyboard

VICE maps the VIC-20 keyboard "positionally" to your PC keyboard — letters / numbers / punctuation type as themselves. VIC-20-specific keys:

KeyVIC-20 key
L CtrlCTRL
L AltCommodore key (C= key — toggles upper/lower case with SHIFT)
Scroll LockRUN/STOP (remapped from Esc — see note below)
Page UpRESTORE (combine with RUN/STOP for warm reset)
F1 / F3 / F5 / F7VIC-20 function keys (only 4 — F2/F4/F6/F8 are SHIFT-modified on hardware)
BackspaceINST/DEL
HomeCLR/HOME
InsertSHIFT + INST/DEL (insert)
TabPound symbol (£)

Why Scroll Lock for RUN/STOP? Browsers capture Esc to exit pointer-lock and fullscreen before the libretro keyboard callback can see it, so a real Esc press never reaches VICE. The bundle remaps Scroll Lock via emulators/_shared/genx-vice-softkeys.js — a document-level keydown listener swallows the Scroll Lock press and dispatches a synthetic Escape event (synthetic events don't trigger the browser's Esc-capture since they aren't real user gestures), which then reaches the core as RUN/STOP. Scroll Lock has no other use in this bundle so the remap is free.

Loading from BASIC

VICE auto-LOADs cartridges and the first program on most disks, but if a game drops you to the READY. prompt, type one of:

CommandEffect
LOAD"$",8List the directory of disk drive 8
LOAD"*",8,1Load + run the first program on the disk
RUNRun the loaded program

Emulator: VICE xvic (GPL-2.0, libretro core) under EmulatorJS (GPL-3.0). Both shipped locally from this site; no network calls at runtime.
