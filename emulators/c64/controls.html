Commodore 64 — Controls

VICE x64 (the gold-standard Commodore 64 emulator) running in your browser via EmulatorJS. The bundled disk image autoboots when the page loads. Click the canvas first so the emulator receives keyboard input.

The initial disk load takes around 30 seconds — that's not a freeze, it's the real-time 1541 drive emulation reading the disk over the IEC bus at period-authentic speed. You'll hear the drive chunk and stepper-click the whole way through; the game starts when it falls silent.

Most C64 games use port 2 by default. If a game ignores Player 1 input, open EmulatorJS's settings menu (gear icon, bottom of the canvas) and swap the joystick port.

C64 keyboard

VICE maps the C64 keyboard "positionally" to your PC keyboard — letters / numbers / punctuation type as themselves. C64-specific keys:

KeyC64 key
L CtrlCTRL
L AltCommodore key (C= key — toggles upper/lower case with SHIFT)
Scroll LockRUN/STOP (remapped from Esc — see note below)
Page UpRESTORE (combine with RUN/STOP for warm reset)
F1 – F8C64 function keys
BackspaceINST/DEL
HomeCLR/HOME
InsertSHIFT + INST/DEL (insert)
TabPound symbol (£)

Why Scroll Lock for RUN/STOP? Browsers capture Esc to exit pointer-lock and fullscreen before the libretro keyboard callback can see it, so a real Esc press never reaches VICE. The bundle remaps Scroll Lock via emulators/_shared/genx-vice-softkeys.js — a document-level keydown listener swallows the Scroll Lock press and dispatches a synthetic Escape event (synthetic events don't trigger the browser's Esc-capture since they aren't real user gestures), which then reaches the core as RUN/STOP. Scroll Lock has no other use in this bundle so the remap is free.

Loading from BASIC

VICE auto-LOADs the first program on most disks, but if a game drops you to the READY. prompt, type one of:

CommandEffect
LOAD"$",8List the directory of disk drive 8
LOAD"*",8,1Load + run the first program on the disk
RUNRun the loaded program
