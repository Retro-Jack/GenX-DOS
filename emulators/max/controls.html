Commodore MAX Machine — Controls

Self-hosted VICE x64 (gold-standard Commodore emulator) running in your browser via EmulatorJS, switched into Ultimax mode — that's the MAX Machine: a Japan-only 1982 home console using the same VIC-II + SID + 6510 chipset as the C64, but with only 2 KB of RAM and no built-in BASIC. Every program is a cartridge. Click the canvas first so the emulator receives keyboard input.

The MAX library is small — about 27 commercial cartridges were ever produced, all in Japan in 1982-83. Eight of the most iconic carts are bundled here, plus the MAX BASIC cartridge (which is how you'd boot the MAX into a usable BASIC environment, since the machine ships without BASIC in ROM).

The MAX has two joystick ports (same DE-9 connector as the C64). Player 1 is on port 1 by default — opposite of the C64 convention. Use the gear icon's joystick port swap if a game ignores Player 1 input.

Why so few games?

The MAX was Commodore's attempt at an entry-level games console for the Japanese market, launched June 1982 — six months before the C64 reached US shelves. With only 2 KB of usable RAM (less than the VIC-20!) and no BASIC ROM, it was crippled out of the gate; the parallel C64 launch made it instantly redundant. Production stopped within months and Commodore Japan moved on. Only ~27 official cartridges ever shipped — the entire library is smaller than a single year of Commodore 64 releases. The MAX is the rarest mainstream Commodore product.

Cart format: Ultimax-mode 8K cartridges mapped to $E000 (and 16K split $8000 + $E000). VICE accepts these as standard .crt files with hardware type 0, EXROM=1, GAME=0. Emulator: VICE x64 (GPL-2.0, libretro core) under EmulatorJS (GPL-3.0). Both shipped locally from this site; no network calls at runtime.
