Commodore 64 — Controls

A Commodore 64 running on Andre Weissflog's chips emulator (a pure-WASM 8-bit emulator collection). The full C64 keyboard + a port-2 joystick are mapped to your PC keyboard.

Joystick (port 2 — Player 1)

KeyAction
↑ ↓ ← →Stick direction
SpaceFire button

Most C64 games use port 2 by default. If a game doesn't respond, it may be looking at port 1 — in chips you can swap via its hidden Dear-ImGui menu (tap the canvas, then look for the menu toggle).

Loading from BASIC (when no autoboot)

If the boot drops to the READY. prompt instead of running a game, type:

CommandEffect
LOAD"$",8List the directory of disk drive 8
LOAD"*",8,1Load + run the first program on the disk
LOAD"FILENAME",8,1Load a specific PRG at its target address
RUNRun the loaded program

Notes

chips C64 auto-detects file format and mounts .d64 images as the 1541 drive, then sends the appropriate LOAD…RUN sequence so most bundled titles start automatically.

For an unmodified C64 boot (no disk loaded), open play.html with no ?game= param — the C64 boots straight to the READY. prompt with BASIC.

Mapping follows the chips C64 (MIT, floooh/chips-test — uses the chips Zlib-licensed 8-bit emulator library by Andre Weissflog). Per-key reference: https://github.com/floooh/chips-test.
