# Changelog

## [Unreleased]

### Changed
- Renamed "Odyssey 2" → "Odyssey²" in all CONSOLE menus (CONSOLE SYSTEMS row 6, MAGNAVOX submenu, and the ODYSSEY² GAMES header). The font sprite sheet is canonical CP437 and the renderer maps JS char codes directly to sprite indices, so a literal `²` (U+00B2 = 178) renders the wrong glyph (CP437 0xB2 = `▓`). The fix uses the same Latin-1↔CP437 trick the box-drawing characters already exploit: store the menu strings with `ý` (U+00FD = 253) which the font renders as the CP437 superscript-2 at sprite-253. Visually wrong in the source, perfect on screen.
- `find` command now treats any query starting with "odyssey" as a match for the Odyssey² entry, regardless of suffix — `find odyssey`, `find "odyssey 2"`, `find "odyssey²"` all hit. Done by truncating the lowercased query to "odyssey" before the walk, so the existing word-boundary matcher still does the work.
- **EmulatorJS framework deduplicated** across the seven VICE-family bundles via `emulators/_shared-ejs/ejs/data/`. Each bundle's per-copy `ejs/` directory (~3.5 MB of framework + ~1.2-1.6 MB core, identical except for the core data file) has been removed; play.html files now point `EJS_pathtodata` at the shared dir via `location.pathname.match(/^.*\/emulators\//) + '_shared-ejs/ejs/data/'`. Cores also deduplicated: c64 + max share `vice_x64-legacy-wasm.data`, plus4 + c16 share `vice_xplus4-legacy-wasm.data`, so the shared `cores/` holds 5 unique files (x64, x128, xvic, xplus4, xpet) totalling ~6.5 MB. Net repo size: 7 × ~3.5 MB = ~25 MB → 1 × ~8.5 MB shared + 7 × <300 KB per bundle = ~12.4 MB. **Saved ~12.6 MB**. The `checkForUpdates()` no-op patch only needs to live in one place now. Preload hints in each play.html updated to `../_shared-ejs/ejs/data/cores/<core>.data` so the browser still parallel-fetches the right core. PET and other keyboard-heavy bundles unaffected by the cores merge since each platform's core is unique.

### Added
- Commodore 128 self-hosted at `emulators/c128/` via EmulatorJS + VICE x128 — **seventh** and final VICE-family sub-system, slotted after Plus/4 in the COMMODRE submenu (C128 shipped Jan 1985, the latest in the family). Configured for **C128 native mode** via `vice_c128_go64: 'disabled'` (BASIC 7.0, 128 KB RAM, dual VIC-IIe + VDC video chips, 8502 + Z80 CPUs) — skipping C64-compat mode (we have a separate c64 bundle for that) and CP/M mode. Per-game `video` field in games.json toggles `vice_c128_video_output` between `VICII` (40-col) and `VDC` (80-col); play.html reads it alongside the existing `model` override pattern from PET. Same unified VICE input config (keyboardInput + numpad joystick). Core: `vice_x128-legacy-wasm.data` (1.6 MB, the largest in the VICE family since it ships ROMs for both video chips + 8502/Z80 CPU). 5 commercial C128-distinctive builds + 5 homebrew 100%-128-only titles: The Last V8 (1985 Mastertronic), Rocky Horror Show (1985 Crystal), Trinity (1986 Infocom 80-col VDC) — later swapped for Knight's Quest, Star Fleet I (1987 Interstel 80-col), Beyond Zork (1987 Infocom 80-col) — later swapped for Wumpus 2.0, plus homebrew 128 Crush, World at War (BASIC 7.0 type-in), Rockfall 128, Phazer, Tetris 128. Sourced from `zimmers.net/anonftp/pub/cbm/c128/games/{40,80}/`. BASIC 7.0 entry uses empty.prg at `\$1C01` (C128 BASIC start address). **CSS scaleY hack** on the canvas works around a VICE x128 quirk where it allocates a 856×288 (2.78:1) framebuffer regardless of `vice_c128_video_output` — RetroArch preserves that aspect inside the 4:3 container, letterboxing top + bottom of the canvas. `transform: scaleY(calc(2.784 / 1.333))` on the canvas + `overflow:hidden` on `#game` stretches the rendered content to fill the 4:3 frame and clips the now-overflowing empty letterbox bars, mimicking how a real C128 monitor stretched VDC's narrow 640×200 framebuffer to fill its 4:3 CRT via non-square pixels.

- `emulators/_shared/genx-vice-softkeys.js` — keyboard remap + soft button for VICE-family RUN/STOP. Browsers capture Esc to exit pointer-lock/fullscreen before the libretro keyboard callback can see it, so a real Esc never reaches VICE. Three layered fallbacks: (1) **Scroll Lock keydown** intercepted at document/window capture phase, dispatches synthetic Escape on document/window/body/canvas — works on Windows + Linux; (2) **Pause/Break** as secondary same path — covers most other PC keyboards; (3) **top-left "RUN/STOP" soft button** (60% opacity at rest, full on hover) positioned out of EJS's bottom toolbar and the top-right controls-link zones — covers macOS (no Scroll Lock key since 2007), ChromeOS, and touch devices. Synthetic events bypass the browser's Esc-capture because they aren't real user gestures. Wired into all 7 VICE play.html files; controls.html on each documents the remap.

- Commodore PET self-hosted at `emulators/pet/` via EmulatorJS + VICE xpet — sixth VICE-family sub-system, slotted at position 1 in the COMMODRE submenu (PET 2001 was Commodore's first computer, Jan 1977, predating VIC-20 by three years). Default model = **3032** (BASIC 2, 32 KB RAM, 40-column, original "graphics" character ROM) — chosen after first-pass 4032 broke 9 of the 10 bundled games with garbled-PETSCII output. Root cause: the BASIC-4 4032 ships a different chargen ROM where 1979-1981-era PETSCII pokes render as different shapes; the 3032's chargen is what those games were authored against. PETSCII Robots is the exception (needs BASIC 4 + 32K + supplies its own TILESET.PET), so it gets a **per-game model override** via a new `model` field in games.json that play.html reads alongside `rom` and sets per-launch. Core file: `vice_xpet-legacy-wasm.data` (1.4 MB, fetched fresh from `cdn.emulatorjs.org/stable` 2025-06-15 build — larger than the other VICE cores because xpet ships ROM sets for all six PET model variants). Drive options: TDE off + autoload warp on (opposite of the C64/VIC-20 chunky-drive-sound recipe) — the PET predates the 1541 era, and the multi-file PETSCII Robots boot chain (PETROBOTS + TILESET.PET + LEVEL-A) is timing-sensitive enough that virtual-fast-load is the safer default. 10 games + BASIC 2 prompt: Android NIM (1979 Commodore), Lunar Lander (1979), Adventureland (1979 Scott Adams), Hangman (1980 Raymond Toy), Space Invaders (1980), Crazy Balloon (1981), ComputerSpace 2001 (1981, Asteroids-flavoured), Frogger (1981), Pac-Man clone (1982), and **PETSCII Robots Shareware** (2022 David Murray / The 8-Bit Guy — the modern survival/strategy title proving the 32K PET still has commercial-grade games being written for it 45 years post-launch). Sourced from `zimmers.net/anonftp/pub/cbm/pet/games/english/` for the classics and `the8bitguy.com/wp-content/uploads/2022/03/Pet-Robots-Shareware-03-29-2022.zip` for PETSCII Robots (the bundled PDF manual stripped per "no docs in bundles"). All 9 classic PRGs load at `\$0401` (PET BASIC start); empty.prg uses `01 04 00 00` for the BASIC entry. Total bundle ~6.3 MB. **Skipped on load-address grounds:** Microchess 2.0 (came wrapped in stale MacBinary header from the zimmers archive — load address `\$0300` cassette buffer load not handled cleanly by EJS autostart); 3D Star Trek + Cosmiads (both load at `\$027A` which is the PET keyboard buffer area — non-standard packaging that VICE doesn't autostart). Could revisit with manual cleanup if any of those titles matter.

- Commodore MAX Machine self-hosted at `emulators/max/` via EmulatorJS + VICE x64 in Ultimax mode — fifth VICE-family sub-system, slotted into the COMMODRE submenu between VIC-20 (1980) and C64 (Aug 1982) since the MAX shipped June 1982 in Japan. Uses the same `vice_x64-legacy-wasm.data` core as the C64 bundle, switched into MAX mode via `EJS_defaultOptions['vice_c64_model'] = 'ultimax'` — that gives 2 KB RAM (no built-in BASIC), period-correct for the real machine. The MAX is the rarest mainstream Commodore product: a Japan-only games console released six months before the C64, killed by the parallel C64 launch within months, ~27 commercial cartridges total. 8 carts bundled + MAX BASIC cartridge as the BASIC entry. Sources: (a) TOSEC `Commodore_Max_Machine_and_Vic_10_TOSEC_2012_04_23` — only 3 entries (Clowns 1982 Commodore/Midway, Omega Race 1982 Midway, Radar Rat Race 1982 Commodore), all raw 8K `.e0` dumps at $E000; (b) Internet Archive `slalom_202210` — 16K Slalom (HAL 1982) raw $8000+$E000 split; (c) extracted from the Multimax EasyFlash compilation (`csdb.dk/release/?id=210760`, Rob Clarke / Michal Pleban 2014, 37 banks total) by hashing + string-fingerprinting the individual game banks — Jupiter Lander (HAL 1982, bank 10), Speed/Bingo Math (Commodore/Bally Midway 1982, bank 31), Billiards (HAL 1983, bank 0, 16K), Pinball Spectacular (HAL 1983, bank 27), MAX BASIC ROM (Commodore 1982, bank 15, 16K). Per-cart-format conversion: wrote a Python wrapper that emits proper Ultimax-mode `.crt` files (hardware type 0, EXROM=1, GAME=0) — raw $E000 8K dumps wrap as a single CHIP packet at $E000; 16K Multimax banks split $8000+$A000 in EasyFlash addressing become $8000+$E000 in Ultimax mode (the $A000 chunks are the cart's $E000 image, banked through EasyFlash's $DE00 latch). Each output `.crt` validates via `file`: e.g. `clowns.crt: Commodore 64 cartridge: "Clowns", UltiMax mode, packet length 0x2010`. Total bundle ~5.4 MB. **Note:** The 4 Multimax-extracted games are bank-string identified but unverified end-to-end in an emulator session — TOSEC + Slalom 4 are canonical, those 4 may need swapping if the bank-game mapping turns out wrong on play.

- VICE keyboard + joystick now coexist across all six CBM bundles (PET / VIC-20 / MAX / C64 / C16 / Plus/4). Earlier attempts toggled `keyboardInput` per game, which forced a choice between joystick (WASD/Ctrl mapped via EJS's gamepad-keymap fallback) and keyboard typing — a no-win for hybrid titles like Last Ninja (joystick movement + F3/F5/Space for items/weapons), Elite (joystick flight + 20-odd letter commands for trading/missiles/jumps), and MAX's Speed/Bingo Math (joystick fire for Bingo + F1 for Speed).
