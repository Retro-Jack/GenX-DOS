EMULATORS` box) and `c:\1.bat` (its dispatcher) collapsed: `1.bat` is gone, the new `menu.bat` directly does `cd emulators\nmenu`. `autoexec.bat` already calls `menu`, so the boot drops straight into the EMULATOR LAUNCHER without the redundant intermediate prompt.

### Changed
- Boot screen rebranded from Award POST to authentic AMIBIOS POST, mirroring the AMIBIOS Configuration table specs that follow. Header `AMIBIOS (C) 1985-1991 American Megatrends Inc.,`; CPU `80486SX`, no numeric coprocessor, memory test `8064 KB OK`; canonical `Hit <DEL> if you want to run SETUP` prompt; AMI BIOS ID footer `40-0102-001102-00101111-121291-i486-K8`. Boot helper renamed `renderAwardPost` → `renderAmiBiosPost`.
- Drive and port specs added to both POST and Configuration table: Floppy A `1.44mb (3.5")`, Floppy B `360kb (5.25")`, Hard Disk C `Type 47`, Hard Disk D `Custom`, Serial Port(s) `COM1`, Parallel Port(s) `LPT1`. POST detection lines aligned at a 27-char label column (widened so `Parallel Port(s)` fits); table value fields padded so all rows remain 80 chars wide.
- POST detection lines now stream in at 500 ms intervals (six lines = 2.5 s of animation) instead of all rendering instantly. The "Hit <DEL>" prompt and BIOS ID footer render after detection finishes. `renderAmiBiosPost(p, onComplete)` now takes a callback; `init()` waits for completion before holding the full POST on-screen for 5 s, then clears and shows the AMIBIOS Configuration table.
- After the AMIBIOS Configuration table renders: two blank lines, `Starting GenX-DOS . . .`, 1 s pause, blank, `Press F1 for help`, blank, then the menu draws below. Closes the boot animation with a "loading the OS" beat before the prompt is interactive.

- Typing `help` now shows the full command reference (content imported from the `Commands` wiki page at `github.com/Retro-Jack/GenX-DOS/wiki/Commands`, reformatted as 80-col plain text — sections: NAVIGATION, FILES, DISPLAY, STUB COMMANDS, KEYBOARD; easter eggs omitted). Replaces the prior one-liner easter-egg `help` response. Boot-end hint reads `Type "help" <enter> for assistance.` (no F1 binding — F1 is unreliable cross-browser since some browsers reserve it for their own help and `preventDefault` on the page can't always suppress that). Implementation: `HELP_TEXT` array + `showFullHelp()` in `commands.js`; `help` registered command calls `showFullHelp()` and `handleCmd`'s trailing `prompt()` re-renders `C:\>` once help finishes.
- 300 ms blank beat after the AMI logo renders before the Tseng video BIOS line speaks up — feels more like a real cold-boot transition (logo + black flash → video BIOS → AMIBIOS POST).
- Authenticity pass on the POST screen: prepended a Tseng Labs ET4000 video BIOS line (`Tseng ET4000 VGA BIOS Version 3.00` + `(C) 1990 Tseng Laboratories, Inc.`) above the AMIBIOS header — real PCs run the video card's BIOS first; animated the memory test as a 4-digit counter incrementing in 64 KB steps every 32 ms (~4 s total to reach `8064 KB OK`); changed `Numeric Coprocessor : Not Installed` → `Numeric Coprocessor : Absent` (canonical AMI wording); reformatted the six device-detection lines to the AMI convention (dropped the `Detecting ` prefix and `... ` filler, e.g. `Floppy Drive A    : 1.44mb (3.5")`); Configuration table updated to match — `Numeric Processor : None` → `Numeric Processor : Absent` and `Display Type : VGA/PGA/EGA` → `Display Type : VGA` (Tseng card is implied by the video BIOS line). Two new boot helpers: `bootMemTestLine` lays out the static prefix + four mutable digit slots + ` KB OK` suffix; `animateMemTest` updates the digit `className`s in place per tick.
- Added a single-page boot animation rendered directly inside `#prompt`: Award BIOS POST → 4 s pause → screen cleared → extended-ASCII AMIBIOS System Configuration table (80-col double-rule outer border, single-rule column divider, two-column hardware fields styled after a 486-era POST screen) → DOS EMULATOR LAUNCHER menu drawn below AMIBIOS (no `cls`, so the table stays on-screen alongside the menu, like a real DOS boot).
