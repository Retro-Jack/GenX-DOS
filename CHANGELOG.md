- New `find <word>` / `find "<phrase>"` command in the virtual DOS prompt. Walks the whole virtual FS recursively from C:\, parses each directory's `menu.bat` to recover the human-readable title for every entry, then matches the query (case-insensitive, word-boundary substring) against both the title and the short code. Reports two sections: `GAMES` (matching `.bat` launchers — files with a `link:` attribute) and `MENUS` (matching sub-directories, with the title pulled from the parent menu's row when present, falling back to the directory name). Each result shows code + title + full path so the user can `CD <path>` and run the code. `[GAMES]` markers stripped from parsed titles. Pure numeric launchers (`1.bat`, `2.bat` — menu shortcuts) and `0. Back` rows excluded from results. Multi-word queries must be wrapped in double quotes (`find "duke nukem"`); bare-spaced input is rejected with an inline usage hint that suggests the quoted form. Registered in `commands.js` and listed in the F1/`help` reference under FILES.
- `serve.sh` — one-shot local-server launcher. Picks port 8765 by default (overridable: `./serve.sh 9000`), checks the port is free, starts `python3 -m http.server`, opens `http://127.0.0.1:8765/prompt/` in the default browser via `xdg-open`/`open`, traps SIGINT/SIGTERM so the server is killed cleanly on Ctrl+C.
